Offscreen canvas demo

OffscreenCanvas allows <canvas> rendering contexts (2D and WebGL) to be used in workers. This can increase parallelism and better performance on multi-core systems.

Support: Behind a flag in Chrome 60. Enable the "Experimental Web Platform Features" flag to use it in about:flags. FF is in development.

Your browser does not support OffscreenCanvas.

What: Select an image below. While the animation is driven by requestAnimationFrame() in the main page, the pixels are manipulated and drawn to the canvas in a web worker! The pixel data is transferred back from the worker by setting up an OffscreenCanvas, calling ctx.transferToImageBitmap(), and sending ImageBitmap back as transferrables using postMessage().
